# GM Mode

NBA trade package optimizer combining a learned valuation model, a SAT feasibility check, and a MIP optimization step. Built for CIS 1921.

## What it does

You pick two NBA teams and the players each side is willing to put on the table. The system finds the subset of those candidates to swap that satisfies the salary-matching rule, the hard cap, no-trade clauses, recently-signed restrictions, and roster-size limits, while maximizing the total trade value across both teams. Value comes from a gradient-boosted regression tree that scores each player against the receiving team's situation (rebuild vs. contention, positional needs).

The solver runs two independent MIP backends (OR-Tools CP-SAT and PuLP/CBC) and prints a delta between their objectives, so you get cross-solver verification for free.

## Setup

You need Python 3.10 or newer.

```
pip install -r requirements.txt
```

That installs Streamlit, scikit-learn, OR-Tools, PuLP, python-sat, pandas, pyarrow, requests, plotly, and nba_api. No API keys, no environment variables, no database.

## Running it

There are two entry points. Both share the same back-end.

**Streamlit UI (the main demo):**

```
python3 -m streamlit run app.py
```

Open the URL it prints (usually http://localhost:8501). On a fresh machine the first load fetches all thirty teams from Basketball-Reference, which takes about twenty-five seconds because BBR rate-limits and we deliberately throttle requests to be polite. Every subsequent run hits the parquet cache under `.cache/` and starts in a couple of seconds.

**CLI demo + benchmark:**

```
python3 main.py
```

This runs the Lakers/Nets demo trade through the full three-layer pipeline, prints the SAT verdict, runs both MIP backends, re-runs with the hard cap turned off to demonstrate the constraint toggles, and finally runs five synthetic benchmark instances generated by the instance generator. Output is plain text suitable for redirecting to a file if you want to save a run.

## File structure

| File | What's in it |
|---|---|
| `main.py` | CLI entry point, runs the full demo and benchmark. |
| `app.py` | Streamlit UI. Picks teams, runs the pipeline, renders results. |
| `data_fetcher.py` | Scrapes Basketball-Reference per-game, advanced, and contract tables, joins them with nba_api roster data, caches to parquet. |
| `valuation_model.py` | Gradient-boosted regression tree player valuation, with team context. |
| `sat_layer.py` | PicoSAT-based feasibility check for boolean CBA constraints. |
| `mip_layer.py` | Both MIP backends (OR-Tools CP-SAT, PuLP/CBC) plus result comparison. |
| `constraints_config.py` | The user-toggleable constraint dataclass. |
| `instance_generator.py` | Parameterized synthetic trade-instance generator for benchmarks. |
| `make_checkin.py` | Generates the April 14 check-in PDF (already submitted). |
| `requirements.txt` | Pinned versions of the libraries above. |
| `.cache/` | Cached parquet files of the BBR dataset, created on first run. |

## How the pieces fit

Data lands in `data_fetcher.py` and gets joined into `PlayerRecord` objects. The valuation model in `valuation_model.py` is trained once per session on two thousand synthetic players (we explain why this is synthetic in the report) and then predicts a value-to-team score for each candidate. The SAT layer enforces the boolean CBA rules and produces a `forced_out` set of player IDs that cannot be traded. That set is passed into the MIP layer, which fixes those variables in advance and then solves the rest of the problem twice (CP-SAT and CBC) under the salary-matching, hard-cap, and non-trivial-trade constraints. Both MIP results come back, and `main.py` prints them side by side along with their objective delta.

The constraint config lives in `constraints_config.py` as a single dataclass. Every layer takes a `ConstraintsConfig` argument so toggles flow through cleanly without being hardcoded anywhere. The Streamlit UI exposes the toggles in the sidebar; the CLI builds them at the top of `main()`.

## Running the benchmark by itself

`instance_generator.py` exposes `generate_benchmark_suite(k, n_teams, n_players_each, salary_variance, constraint_tightness, base_seed)` which returns `k` random `TradeInstance` objects with reproducible seeds. `main.py` Section 7 already calls this with `k=5`. To sweep larger numbers of instances or different tightness values, edit the parameters in `main()` directly. The constraint-tightness experiment described in the report was a manual sweep with a small driver script that loops over tightness from 0 to 1.0 in steps of 0.1.

## Common issues

If `streamlit run app.py` fails with `File does not exist`, you are running it from the wrong directory. `cd` into the project root first.

If the first load hangs on Basketball-Reference, your network may be blocking requests to www.basketball-reference.com. The fallback `_DEMO_PLAYERS` pool in `data_fetcher.py` is enough to run `main.py` end to end.

If `pysat` will not install on your machine (it sometimes fails on macOS without command-line tools), the SAT layer falls back to a procedural Python check that enforces the same rules. The CLI prints a notice when this happens.
